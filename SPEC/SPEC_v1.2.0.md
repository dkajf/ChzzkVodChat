# ChzzkVodChat v1.2.0 Specification

## AI 개발 지침

이 문서는 Version 1.2의 최종 명세이다.

구현 과정에서 명세를 임의로 변경하거나 새로운 기능을 추가하지 않는다.

모호한 사항은 기존 코드 스타일과 명세를 우선하며, 필요한 경우 사용자에게 확인 후 구현한다.

구현 전에는 반드시 SPEC.md 전체를 읽고 구현한다.

---

## 프로젝트 개요

CHZZK VOD 채팅 수집 및 분석 프로그램.

Version 1.1은 채팅 수집 기능을 구현하였다.

Version 1.2는 기존 수집 기능을 유지하면서 채팅 분석 기능을 추가한다.

기존 기능은 가능한 수정하지 않고, 새로운 함수를 추가하는 방식으로 개발한다.

---

# 구현 원칙

## 코드 구조

* Version 1.1 코드를 기반으로 개발한다.
* 기존 함수는 가능한 수정하지 않고 새로운 함수를 추가한다.
* 수집 기능과 분석 기능은 서로 독립적으로 유지한다.
* 함수 하나는 하나의 역할만 수행하도록 작성한다.
* 기존 출력 스타일과 코드 스타일을 유지한다.

# Version 1.2 목표

# 입력 규칙

## 기본 규칙

* 입력은 최대 10개까지 등록할 수 있다.
* 입력은 위에서부터 순서대로 처리한다.
* 빈 입력(Enter)은 입력 종료를 의미한다.
* 빈 입력 이후에는 추가 입력을 받지 않는다.

---

## 모드 결정

프로그램은 **첫 번째 형식이 올바른 입력**을 기준으로 실행 모드를 결정한다.

실제 존재 여부는 모드 결정에 영향을 주지 않는다.

지원하는 모드는 다음 두 가지이다.

* URL 수집 모드
* 채팅 분석 모드

첫 번째 형식이 올바른 입력이 URL이면 URL 수집 모드로 동작한다.

첫 번째 형식이 올바른 입력이 파일명이면 채팅 분석 모드로 동작한다.

모드가 결정된 이후에는 다른 형식의 입력은 모두 무시한다.

예시

```
URL
URL
파일명
```

→ URL 모드

```
파일명
URL
파일명
```

→ 분석 모드

---

## JSON 형식

Version 1.2는 Version 1.1에서 생성한 JSON 파일 형식을 그대로 사용한다.

JSON 구조는 변경하지 않는다.

---

## 인코딩 방식

모든 입출력 파일은 UTF-8 인코딩을 사용한다.

---

## 형식이 올바른 입력

### URL

다음 형식의 CHZZK VOD URL을 지원한다.

```
https://chzzk.naver.com/video/123456
```

URL 형식이 올바르면 유효한 입력으로 판단한다.

실제 존재하는 VOD인지는 이후 단계에서 확인한다.

---

### 파일명

다음 형식을 지원한다.

```
2026-06-24_123456
2026-06-24_123456.json
2026-06-24_123456.txt
```

확장자는 자동으로 제거한다.

파일명 형식이 올바르면 유효한 입력으로 판단한다.

실제 JSON 파일 존재 여부는 이후 단계에서 확인한다.

---

## 형식이 잘못된 입력

다음 입력은 형식이 올바르지 않은 입력으로 처리한다.

* 지원하지 않는 URL
* 지원하지 않는 파일명
* 의미 없는 문자열
* 특수문자만 입력한 경우

형식이 잘못된 입력은 모드 결정에 사용하지 않는다.

안내 메시지를 출력한 뒤 다음 입력을 계속 처리한다.

예시

```
ㅎㅎㅎ
???
abc
URL
```

→ 첫 번째 형식이 올바른 입력은 URL

→ URL 모드

---

## 중복 입력

동일한 URL은 한 번만 처리한다.

동일한 파일명도 한 번만 처리한다.

중복 입력은 안내 메시지를 출력한 뒤 무시한다.

---

## 최대 입력 개수

최대 10개까지 등록할 수 있다.

11번째 입력부터는 받지 않는다.

최대 개수에 도달하면 입력을 종료한다.

---

## 모드 결정 이후 검증

URL 모드에서는 각 URL의 실제 VOD 존재 여부를 확인한다.

분석 모드에서는 각 파일명의 JSON 파일 존재 여부를 확인한다.

조회에 실패한 대상은 "조회 실패" 메시지를 출력하고 건너뛴다.

다른 입력은 계속 처리한다.

---

## 모든 대상이 조회 실패한 경우

모드는 유지한다.

단, 실제 처리 가능한 대상이 하나도 없는 경우에는 안내 메시지를 출력하고 프로그램을 종료한다.


# 출력 규칙

## 기본 규칙

* 분석 결과는 하나의 `_analysis.txt` 파일에 저장한다.
* 기존 JSON 및 TXT 파일은 수정하지 않는다.
* 분석 결과는 항상 동일한 형식을 유지한다.
* 모든 시간은 `HH:MM:SS` 형식으로 출력한다.
* 모든 수치는 사람이 읽기 쉬운 형태로 출력한다.

---

## 출력 순서

분석 결과는 다음 순서를 따른다.

1. 분석 정보
2. 채팅량 TOP10
3. 증가율 TOP10
4. 키워드 TOP10
5. 상세로그

출력 순서는 항상 동일하다.

---

## 분석 정보

다음 정보를 출력한다.

* 파일명
* 총 채팅 수
* 전체 평균 채팅 수
* 분석 키워드

---

## TOP 출력

각 TOP은 최대 10개까지 출력한다.

출력 순서는 높은 값부터 내림차순이다.

동일한 값이 발생하면 더 이른 시간 먼저 출력한다.

TOP 출력에는 다음 정보를 포함한다.

* 시간
* 채팅 수
* 키워드 포함 채팅 수
* 키워드 비율
* 증가율
* 전체 평균 대비 비율

---

## 키워드 검색

키워드는 부분 문자열 포함 여부로 판단한다.

예)

키워드 : ㅋ

"ㅋ"
"ㅋㅋ"
"ㅋㅋㅋㅋㅋㅋ"

→ 모두 포함

키워드 : 미친

"미친"

"진짜미친ㅋㅋ"

→ 모두 포함

Version 1.2에서는 대소문자 구분 및 정규식 검색은 지원하지 않는다.

---

## 평균대비

평균대비 계산시 소수 첫째 자리에서 반올림하여 정수로 출력한다.

---

## 상세로그

방송 시작부터 종료까지 모든 분을 시간순으로 출력한다.

채팅이 없는 분도 출력한다.

상세로그에는 다음 정보를 포함한다.

* 시간
* 채팅 수
* 키워드 포함 채팅 수
* 키워드 비율
* 증가율

상세로그에는 전체 평균 대비 비율을 출력하지 않는다.

---

## 시간 출력

시간은 항상 1분 단위 기준으로 출력한다.

예)

00:00:00

00:01:00

00:02:00

...

---

## 채팅 수

채팅 수는 해당 1분 동안 발생한 전체 채팅 개수이다.

천 단위 구분기호(,)를 사용한다.

예)

421

1,243

12,538

---

## 키워드 수

키워드 수는 키워드를 포함한 채팅 개수이다.

한 채팅에 키워드가 여러 번 포함되어 있어도 1회만 계산한다.

예)

"ㅋㅋㅋㅋㅋㅋ"

→ 키워드 포함 채팅 1개

---

## 키워드 비율

공식

(키워드 포함 채팅 수 ÷ 전체 채팅 수) × 100

소수점 첫째 자리까지 출력한다.

예)

43.4%

50.0%

100.0%

---

## 이동평균

±10분

현재 분 제외

존재하는 데이터만 평균

채팅이 0인 분도 평균 계산 대상

---

## 전체평균

전체 평균은

방송 전체 분 수 기준

(채팅 0인 분 포함)

으로 계산한다.

---

## 증가율

공식

(현재 채팅량 - 이동평균)
÷ 이동평균 × 100

소수점은 출력하지 않는다.

반올림하여 정수로 출력한다.

예)

124%

-37%

0%

---

## 증가율 강조

증가율 50%마다 ▲ 하나를 추가한다.

예)

5%

→ (아이콘 없음)

68%

→ ▲68%

124%

→ ▲▲124%

176%

→ ▲▲▲176%

249%

→ ▲▲▲▲249%

음수에는 ▲를 출력하지 않는다.

예)

-42%

0%

---

## 전체 평균 대비

TOP 출력에서만 사용한다.

공식

현재 채팅 수 ÷ 전체 평균 채팅 수 × 100

소수점은 출력하지 않는다.

반올림하여 정수로 출력한다.

예)

198%

84%

---

## 빈 데이터

채팅이 없는 분도 상세로그에 출력한다.

예)

[03:24:00]

채팅 0 | 키워드 0 (0.0%) | 0%

---

## 출력 형식

Version 1.2에서는 출력 형식을 변경하지 않는다.

항목의 순서와 줄바꿈은 출력 UI 템플릿을 기준으로 구현한다.

# Version 1.2 완료 조건

Version 1.2는 아래 조건을 모두 만족하면 완료된 것으로 판단한다.

* URL 수집 기능이 Version 1.1과 동일하게 동작한다.
* 파일명 입력 시 JSON을 자동 검색한다.
* 분석 결과를 `_analysis.txt`로 저장한다.
* 출력 형식이 출력 UI 템플릿과 동일하다.
* 채팅량 TOP10이 출력된다.
* 증가율 TOP10이 출력된다.
* 키워드 TOP10이 출력된다.
* 상세로그가 출력된다.
* 기존 Version 1.1 기능이 정상 동작한다.

## Version 1.2 출력 원칙

Version 1.2는 데이터를 분석하여 제공하는 기능만 수행한다.

TOP 목록은 참고 자료이며, 자동으로 하이라이트를 선정하거나 결과를 필터링하지 않는다.

최종 판단은 사용자가 수행한다.

명세에 없는 기능은 구현하지 않는다.

성능 최적화나 리팩토링은 동작이 동일한 경우에만 수행한다.
