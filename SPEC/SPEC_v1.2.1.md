# Spec v1.2.1

## 목적

v1.2.1은 새로운 기능을 추가하는 버전이 아니다.

이번 버전의 목표는 기존 분석 기능의 **신뢰도 향상**과 **콘솔 UI 개선**이다.

---

# 1. 알고리즘 개선

## 1.1 최소 채팅량 필터

### 목적

채팅 수가 매우 적은 구간에서 발생하는 허수를 제거한다.

예)

* 새벽 시간대
* 방송 종료 직전
* 휴식 시간

위와 같은 구간은 채팅 수가 적기 때문에 증가율과 키워드 비율이 과도하게 계산되는 문제가 있다.

### 변경

* 최소 채팅 수 기준을 추가한다.
* 기준 미만의 구간은 증가율 및 키워드 순위 계산에서 제외한다.
* 기준값은 코드 내부 상수로 관리한다.
* 사용자 설정 기능은 v1.3에서 구현한다.

---

## 1.2 전체 평균 → 이동평균

### 기존

전체 방송 평균과 비교

```text
평균 대비 650%
```

### 문제

* 방송 초반
* 피크 시간
* 새벽 시간

등 시간대에 따른 기본 채팅량 차이를 반영하지 못한다.

### 변경

최근 N분 이동평균을 기준으로 비교한다.

예)

```text
최근 15분 평균 : 58

현재 : 82

=> 이동평균 대비 +41%
```

이를 통해 방송 상황에 따라 기준이 자연스럽게 변화하도록 한다.

---

## 1.3 키워드 TOP 분리

기존

```text
키워드 TOP10
```

↓

변경

```text
키워드 개수 TOP10

키워드 비율 TOP10
```

### 목적

* 키워드가 많이 나온 구간
* 키워드 밀도가 높은 구간

을 각각 확인할 수 있도록 한다.

---

## 1.4 키워드 비율 시각화

숫자만 출력하지 않고 단계별 특수기호를 함께 출력한다.

### 기준

| 비율       | 표시  |
| -------- | --- |
| 10% 미만   | -   |
| 10~19.9% | ▲   |
| 20~29.9% | ▲▲  |
| 30% 이상   | ▲▲▲ |

예)

```text
키워드 20 (15.5%) ▲
키워드 30 (28.3%) ▲▲
키워드 25 (34.2%) ▲▲▲
```

---

## 1.5 이동평균 증가율 시각화

기존에는 증가율에 ▲를 무제한 출력하였다.

예)

```text
▲▲▲▲▲▲▲▲515%
```

이는 가독성이 떨어지므로 키워드 비율과 동일한 방식으로 단순화한다.

### 기준(초안)

| 이동평균 대비 증가율 | 표시  |
| ----------- | --- |
| 30% 미만      | -   |
| 30~59.9%    | ▲   |
| 60~99.9%    | ▲▲  |
| 100% 이상     | ▲▲▲ |

(기준값은 구현 후 테스트를 통해 조정 가능)

# 2. 콘솔 UI 개선

## 2.1 분석 대상 확인 화면

현재 출력이 길어 CMD 기본 크기에서 확인이 어렵다.

파일당 1줄만 사용하도록 변경한다.

### 예시

```text
====================================
CHZZK VOD CHAT ANALYZER
====================================

[분석 대상]

01. 2026-05-20_1234567
02. 2026-05-21_1234568
03. 2026-05-22_1234569
04. 2026-05-23_1234560

총 파일 : 4개
키워드 : ㅋ

====================================
채팅 분석을 시작하시겠습니까? (Y/N)
====================================
>
```

---

## 2.2 수집 대상 확인 화면

URL 대신 방송 정보를 표시한다.

표시 형식

```text
날짜 | 방송 제목 | VOD 번호
```

예시

```text
====================================
CHZZK VOD CHAT COLLECTOR
====================================

[수집 대상]

01. 2026-06-25 | POE2 하드코어... | 13298190
02. 2026-06-24 | 엘든링 DLC...    | 13298412
03. 2026-06-23 | 잡담 방송        | 13298635

총 대상 : 3개

====================================
채팅 수집을 시작하시겠습니까? (Y/N)
====================================
>
```

제목이 길 경우 적절히 생략(...) 처리한다.

---

## 2.3 확인 UI 통일

수집 모드와 분석 모드 모두

```text
(Y/N)
```

확인 방식으로 통일한다.

---

## 2.4 분석 완료 메뉴

분석 완료 후 프로그램이 종료되지 않도록 변경한다.

```text
====================================
분석이 완료되었습니다.
====================================

1. 처음으로 돌아가기
2. 종료

선택 :
```

이를 통해 프로그램 재실행 없이 연속 분석이 가능하도록 한다.

---

## 2.5 출력 정렬 개선

콘솔 출력 시 모든 컬럼이 일정한 위치에 출력되도록 문자열 폭을 지정한다.

Python f-string의 Width Formatting을 사용하여 구현한다.

예)

```python
f"{value:>3}"
```

등을 활용하여 모든 행에서 `|` 구분자가 동일한 위치에 출력되도록 구현한다.

---

### 출력 예시

```text
채팅 120 | 키워드  34 (28.3%) ▲▲  | 평균 +92%  ▲▲
채팅  73 | 키워드  25 (34.2%) ▲▲▲ | 평균 +12%  -
채팅  53 | 키워드  21 (39.6%) ▲▲▲ | 평균 +108% ▲▲▲
채팅  27 | 키워드   2 ( 7.4%) -   | 평균 -18%  -
```
