### 5-3 기획 중 파악한 취약점

#### TC-009

- 항목

    마지막 채팅 페이지 처리

- 배경

    치지직 API는 마지막 데이터 수집 이후 종료 처리가 정상적으로 이루어져야 한다.

- 예상 결과

    중복 수집 없이 정상 종료된다.

- 결과

    PASS

#### TC-010

- 항목

    채팅이 적은 방송

- 배경

    채팅 수가 적은 방송에서도 오류 없이 종료되어야 한다.

- 예상 결과

    정상 저장 후 종료한다.

- 결과

    PASS

#### TC-011

- 항목

    50개 단위 채팅 수집

- 배경

    치지직 API는 한 번의 요청으로 최대 50개의 채팅 데이터를 반환한다.

- 목적

    데이터의 무결성을 중점으로 본다
    모든 페이지를 순차적으로 조회하여 채팅 데이터가 누락 없이 저장되는지 확인한다.

- 예상 결과

    모든 채팅이 순차적으로 저장된다.

- 결과

    PASS

#### TC-012

- 항목

    페이지 전환(next_time)

- 배경

    다음 페이지 조회를 위해 next_time 값을 사용한다.

- 목적

    next_time 값이 정상적으로 전달되어 다음 채팅 데이터를 이어서 조회하는지 확인한다.

- 예상 결과

    페이지가 정상적으로 이어지며 데이터가 누락되지 않는다.

- 결과

    PASS
