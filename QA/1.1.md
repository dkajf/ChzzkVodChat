# CHZZK VOD Chat v1.1 개발 이력

## 1. 버전 개요

### 버전

v1.1

### 개발 목적

v1.0 배포 후 실제 사용 과정에서 발견된 사용성 및 안정성 문제를 개선하고, 프로그램의 신뢰성을 향상시키는 것을 목표로 한다.

이번 버전에서는 새로운 핵심 기능을 추가하기보다 기존 기능의 완성도를 높이는 데 중점을 두었다.

---

# 2. 개선 배경

v1.0 테스트 및 실제 사용 과정에서 다음과 같은 개선 요구사항이 확인되었다.

* 입력값 검증 부족
* 예외 상황 발생 시 원인 확인 어려움
* 파일 저장 과정의 안정성 부족
* 콘솔 출력 정보 부족
* 장시간 수집 시 진행 상황 파악 어려움

이에 따라 사용자 편의성과 안정성을 향상시키기 위한 개선 작업을 진행하였다.

---

# 3. 개선 사항

## 3.1 입력값 검증 강화

### 개선 전

* URL 형식이 올바르지 않아도 프로그램이 그대로 실행을 시도하였다.

### 개선 후

* 입력값을 사전에 검증하여 잘못된 URL인 경우 사용자에게 안내 메시지를 출력하도록 수정하였다.

---

## 3.2 예외 처리 개선

### 개선 전

예외 발생 시 프로그램이 종료되거나 원인을 파악하기 어려웠다.

### 개선 후

예외 상황에 대한 메시지를 출력하여 사용자가 오류 원인을 쉽게 확인할 수 있도록 개선하였다.

---

## 3.3 파일 저장 안정성 향상

### 개선 전

방송 제목을 파일명으로 사용할 때 저장 과정의 안정성이 부족하였다.

### 개선 후

파일 저장 로직을 개선하여 저장 실패 가능성을 줄였다.

---

## 3.4 진행 상황 출력 개선

### 개선 전

현재 얼마나 수집되었는지 직관적으로 확인하기 어려웠다.

### 개선 후

채팅 수집 진행 상황과 누적 수집 개수를 보다 명확하게 출력하도록 개선하였다.

---

## 3.5 로그 출력 정리

### 개선 전

콘솔 출력 정보가 일정하지 않아 현재 상태를 파악하기 어려웠다.

### 개선 후

출력 형식을 통일하여

* 시작
* 진행
* 완료
* 오류

상태를 쉽게 구분할 수 있도록 개선하였다.

# 9. 실제 사용 중 확인한 개선 사항

이번 테스트 과정에서 다음 사항을 차기 버전 개선 대상으로 기록하였다.

* 장시간 방송에서 동일 이벤트가 여러 분에 걸쳐 반복 추천되는 문제
* 채팅량이 적은 구간이 증가율만으로 상위에 노출되는 현상
* 채팅량, 증가율, 키워드를 각각 따로 분석하여 실제 하이라이트 선정 효율이 떨어지는 문제
* 분석 결과에서 연속된 이벤트를 하나의 구간으로 병합할 필요성
* 이동 평균을 활용한 증가율 계산 방식 검토 필요

# 11. 테스트 결론

v1.1은 기존 핵심 기능을 유지하면서 데이터 수집 안정성과 사용성을 향상시킨 버전이다.

Regression Test를 통해 기존 기능이 정상적으로 유지됨을 확인하였으며, Data Integrity Test를 통해 치지직 API의 페이지 기반 채팅 수집 과정에서 데이터의 중복 및 누락이 발생하지 않음을 검증하였다.

또한 실제 장시간 VOD를 이용한 사용성 테스트를 수행한 결과, 하이라이트 분석 기능의 정확도를 더욱 높이기 위해 이벤트 점수 기반 분석과 연속 이벤트 병합 기능을 차기 버전(v1.2)의 개선 과제로 선정하였다.
