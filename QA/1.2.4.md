# CHZZK VOD Chat Analyzer v1.2.4 개발 이력

## 1. 버전 개요

### 버전

v1.2.4

### 개발 목적

v1.2.4는 기존 채팅 수집 및 분석 기능을 유지하면서, 설정 파일 기반의 옵션 관리와 기본 cmd 환경에서의 사용성을 개선하는 것을 목표로 한다.

이번 버전에서는 프로그램 실행 시 설정 파일을 자동으로 생성하고, 잘못된 설정값을 안전하게 복구하도록 구성하였다. 또한 분석 파일 입력 정책을 명확히 하여 `.txt` 파일명 입력 시에도 같은 이름의 `.json` 파일을 찾아 분석하도록 안내하고, 손상된 JSON이나 잘못된 분석 파일이 입력되어도 프로그램이 종료되지 않도록 예외 처리를 강화하였다.

---

# 2. 개발 배경

기존 버전은 최대 URL 수, 이동 평균 기준, TOP 출력 개수 등 주요 값이 코드에 고정되어 있어 사용자가 실행 환경에 맞게 조정하기 어려웠다.

또한 기본 Windows cmd 크기에서 일부 안내 문구와 진행률 출력이 길게 표시되어 가독성이 떨어질 수 있었고, 분석 모드에서 손상된 JSON 파일을 읽으면 프로그램이 종료될 가능성이 있었다.

이에 따라 설정 파일 시스템을 추가하고, 실행 중 오류가 발생해도 메인 메뉴로 복귀할 수 있도록 안정성을 개선하였다.

---

# 3. 개선 사항

## 3.1 설정 파일 시스템 추가

### 개선 전

* 주요 옵션이 코드 상수로 고정되어 있었다.
* 사용자가 최대 VOD 수, 기본 키워드, TOP 개수 등을 쉽게 조정할 수 없었다.

### 개선 후

* 실행 폴더에 `CHZZK_VOD_CHAT_Settings.txt`를 자동 생성한다.
* 설정 생성, 읽기, 포맷 검증, 값 검증을 독립 함수로 분리하였다.
* 프로그램 시작 시 설정을 한 번만 읽고 이후 메모리 설정값을 사용한다.
* 잘못된 설정값은 기본값으로 복구하고 최신 포맷으로 다시 저장한다.

---

## 3.2 설정 검증 및 복구 강화

### 개선 전

* 설정 파일이 없었으므로 사용자가 조정할 수 있는 검증 대상이 없었다.

### 개선 후

* `ConfigFormat=1`을 기준으로 설정 파일 포맷을 검증한다.
* 누락, 잘못된 자료형, 허용 범위 초과, 빈 문자열, 잘못된 Boolean 값을 복구한다.
* 설정 파일이 손상되거나 읽을 수 없으면 최신 형식으로 재생성한다.
* 생성, 초기화, 재생성 시 사용자에게 안내 메시지를 출력한다.

---

## 3.3 cmd 친화 UI 개선

### 개선 전

* 메인 메뉴 안내와 진행률 출력이 길어 기본 cmd 폭에서 읽기 불편할 수 있었다.
* 분석 결과의 TOP 행이 길게 표시되었다.

### 개선 후

* 메인 메뉴 안내를 짧고 명확하게 정리하였다.
* 진행률 출력은 `% | 채팅 수 | 시간` 형태로 축약하였다.
* TOP 결과 행은 `채팅 | 키워드 | 비율 | 증가` 형태로 압축하였다.
* 수집 및 분석 완료 후 결과 파일 경로를 더 명확히 출력한다.

---

## 3.4 분석 파일 입력 안내 개선

### 개선 전

* `.txt` 파일명을 입력할 수 있었지만 실제로는 같은 이름의 `.json` 파일만 분석한다는 점이 명확하지 않았다.

### 개선 후

* 메인 메뉴에서 `.json` 또는 `.txt` 이름을 입력할 수 있음을 안내한다.
* `.txt` 입력 시에도 실제 분석에는 같은 이름의 `.json` 파일이 필요하다는 점을 안내한다.
* `.txt` 내용 자체는 분석하지 않고, JSON 원본을 분석 데이터로 사용한다.

---

## 3.5 분석 JSON 예외 처리 강화

### 개선 전

* 비어 있는 JSON 파일, 손상된 JSON, 예상 구조가 아닌 JSON을 입력하면 프로그램이 종료될 수 있었다.

### 개선 후

* JSON 로딩과 구조 검증을 `load_analysis_chats()` 및 `validate_chat_json()`으로 분리하였다.
* 빈 파일, 빈 목록, 잘못된 JSON 형식, 필수 필드 누락을 감지한다.
* 오류 발생 시 사유를 출력하고 프로그램을 종료하지 않은 채 메인 메뉴로 복귀한다.

---

## 3.6 분석 파일 검색 최적화

### 개선 전

* 분석 파일을 찾을 때 입력값마다 `chat` 폴더를 순회할 수 있었다.

### 개선 후

* `chat` 폴더의 `.json` 파일을 한 번만 인덱싱한 뒤 파일명을 조회한다.
* 파일 수가 많아져도 분석 대상 조회가 더 안정적으로 동작한다.

## 1. 테스트 목적

v1.2.4의 설정 파일 시스템, cmd 친화 UI, 분석 파일 입력 안내, JSON 예외 처리가 정상적으로 동작하는지 확인한다. 또한 기존 채팅 수집 및 분석 기능이 유지되는지 검증한다.

---

## 2. 테스트 범위

### Regression Test

* 기존 VOD 채팅 수집 기능 유지 여부
* 기존 JSON 기반 분석 기능 유지 여부
* 기존 분석 결과 파일 생성 여부

### Functional Test

* 설정 파일 자동 생성
* 설정 파일 포맷 검증
* 잘못된 설정값 복구
* 설정값 기반 옵션 적용
* `.txt` 파일명 입력 시 같은 이름의 `.json` 파일 조회

### Validation Test

* 빈 JSON 파일 입력
* 손상된 JSON 파일 입력
* 빈 채팅 목록 입력
* 필수 필드가 누락된 JSON 입력
* `OutputFolder`가 존재하지 않는 경우

### Usability Test

* 메인 메뉴 안내 가독성
* 진행률 출력 가독성
* 완료 메시지와 결과 파일 경로 표시

## 3. 테스트 케이스

* 테스트 항목
  * 설정 파일이 없는 상태에서 프로그램 실행

  * `CHZZK_VOD_CHAT_Settings.txt`가 생성된다.
  * 생성 안내가 출력된다.

### TC-702: 설정 파일 포맷 불일치 복구

* 테스트 항목
  * `ConfigFormat` 값이 다른 설정 파일로 실행
* 예상 결과
  * 최신 설정 형식으로 초기화된다.
  * 초기화 사유가 출력된다.

### TC-703: 일부 설정값 오류 복구

* 테스트 항목
  * `MaxVods=abc`, `MovingAverageMinutes=100` 등 잘못된 값 입력
* 예상 결과
  * 잘못된 항목만 기본값으로 복구된다.
  * 복구 사유가 출력된다.
  * 설정 파일은 최신 포맷으로 저장된다.

### TC-704: 설정값 기반 수집 옵션 적용

* 테스트 항목
  * `MaxVods` 값을 변경한 뒤 URL 입력
* 예상 결과
  * 변경된 최대 URL 수가 메인 메뉴와 입력 제한에 반영된다.

### TC-705: 설정값 기반 분석 옵션 적용

* 테스트 항목
  * `DefaultKeyword`, `TopCount`, `MovingAverageMinutes` 값 변경 후 분석
* 예상 결과
  * 변경된 설정값이 분석 안내와 결과에 반영된다.

### TC-706: `.txt` 파일명 입력 안내 확인

* 테스트 항목
  * `YYYY-MM-DD_영상번호.txt` 형식 입력
* 예상 결과
  * 같은 이름의 `.json` 파일을 찾아 분석한다.
  * `.json`이 없으면 JSON 파일이 필요하다는 안내가 출력된다.

### TC-707: 빈 JSON 파일 처리

* 테스트 항목
  * 내용이 없는 `.json` 파일 입력
* 예상 결과
  * JSON 형식 오류 안내가 출력된다.
  * 프로그램이 종료되지 않고 메인 메뉴로 복귀한다.

### TC-708: 빈 채팅 목록 처리

* 테스트 항목
  * `[]` 내용의 `.json` 파일 입력
* 예상 결과
  * 채팅 데이터가 비어 있다는 안내가 출력된다.
  * 프로그램이 종료되지 않고 메인 메뉴로 복귀한다.

### TC-709: 잘못된 JSON 구조 처리

* 테스트 항목

* 예상 결과
  * 누락된 필드 안내가 출력된다.
  * 프로그램이 종료되지 않고 메인 메뉴로 복귀한다.

### TC-710: cmd 출력 가독성 확인

* 테스트 항목
  * 기본 Windows cmd 크기에서 메뉴, 진행률, 결과 출력 확인
* 예상 결과
  * 핵심 안내가 짧게 표시된다.
  * 진행률과 TOP 결과가 과도하게 길지 않게 표시된다.

### TC-711: 분석 파일 검색 최적화 확인

  * 분석 대상 조회가 정상적으로 수행된다.
  * 같은 이름의 JSON 파일을 안정적으로 찾는다.

## 4. 예상 결과

* 설정 파일이 없어도 프로그램이 바로 사용 가능하다.
* 잘못된 설정이나 분석 파일로 인해 프로그램이 종료되지 않는다.
* 사용자는 메모장으로 주요 옵션을 쉽게 수정할 수 있다.
* 기본 cmd 환경에서 안내와 결과 출력의 가독성이 향상된다.
* 기존 채팅 수집 및 분석 기능은 유지된다.

# 6. 향후 개발 계획

v1.2.4 이후에는 설정 가능한 분석 옵션을 더 세분화하고, 분석 결과의 요약성과 시각적 가독성을 높이는 방향을 검토한다.

예정 기능

* 다중 키워드 분석 지원
* 분석 결과 요약 리포트 자동 생성
* 설정 파일 항목 확장
* TOP 구간 시각화 기능
