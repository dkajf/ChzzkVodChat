# CHZZK VOD Chat Analyzer v1.2.1 개발 이력

## 1. 버전 개요

### 버전

v1.2.1

### 개발 목적

v1.2에서 구현한 채팅 분석 기능을 실제 치지직 VOD에 적용하여 분석 결과를 검증하고, 분석 정확도와 활용성을 향상시키기 위한 개선 작업을 진행하였다.

이번 버전에서는 새로운 기능을 대규모로 추가하기보다 실제 사용 데이터를 기반으로 분석 알고리즘을 개선하고, 결과의 신뢰성을 높이는 데 중점을 두었다.

---

# 2. 개발 배경

v1.2에서 채팅량, 증가율 및 키워드 분석 기능을 구현한 이후 실제 방송 10개 이상을 분석하여 결과를 검증하였다.

검증 과정에서 다음과 같은 문제점을 확인하였다.

* 증가율이 높은 작은 이벤트가 상위에 추천되는 현상
* 동일 이벤트가 여러 분에 걸쳐 중복 추천되는 현상
* 분석 결과를 실제 편집 구간으로 활용하기 어려운 경우 발생
* 채팅량, 증가율 및 키워드를 각각 독립적으로 분석하여 종합적인 판단이 어려움

이에 따라 분석 알고리즘의 품질을 개선하기 위한 작업을 진행하였다.

---

# 3. 개선 사항

## 3.1 실제 방송 데이터 기반 검증

### 개선 전

기능 동작 여부를 중심으로 테스트를 수행하였다.

### 개선 후

10개 이상의 실제 치지직 VOD를 분석하여 다양한 방송 환경에서 결과를 검증하였다.

---

## 3.2 추천 결과 분석

### 개선 전

채팅량, 증가율 및 키워드 결과를 각각 확인하였다.

### 개선 후

추천된 구간을 실제 방송과 비교하여 추천 품질을 검토하고 개선이 필요한 요소를 도출하였다.

---

## 3.3 분석 알고리즘 개선

### 개선 전

채팅량, 증가율 및 키워드를 개별적으로 분석하였다.

### 개선 후

각 분석 방식의 장단점을 비교하고 향후 이벤트 점수 기반 분석을 위한 개선 방향을 설계하였다.

---

## 3.4 동일 이벤트 중복 추천 분석

### 개선 전

동일한 이벤트가 여러 분에 걸쳐 반복 추천되는 현상이 발생하였다.

### 개선 후

연속된 추천 구간을 하나의 이벤트로 관리하는 개선 방향을 수립하였다.

---

## 3.5 증가율 계산 방식 검토

### 개선 전

직전 1분 대비 증가율을 계산하였다.

### 개선 후

실제 데이터를 분석한 결과 작은 채팅량에서도 높은 증가율이 발생하는 문제를 확인하였으며, 이동 평균 기반 계산 방식의 필요성을 도출하였다.

# 5. 테스트를 통해 확인한 주요 이슈

## AI-001

증가율만 높은 작은 이벤트가 실제 하이라이트보다 우선 추천되는 현상

---

## AI-002

동일 이벤트가 여러 분에 걸쳐 반복 추천되는 현상

---

## AI-003

채팅량이 높은 구간과 웃음 반응이 많은 구간이 서로 다르게 추천되는 현상

---

## AI-004

분석 결과가 실제 편집 구간과 완전히 일치하지 않는 사례 존재

---

## AI-005

단일 키워드 기반 분석으로 다양한 채팅 반응을 충분히 반영하지 못함

---

# 6. 개선 효과

이번 개선을 통해 다음과 같은 성과를 얻었다.

* 실제 방송 데이터를 기반으로 분석 알고리즘을 검증하였다.
* 분석 결과와 실제 하이라이트의 차이를 확인하였다.
* 분석 알고리즘의 개선 방향을 명확히 정의하였다.
* 차기 버전 개발을 위한 요구사항을 도출하였다.

---

# 7. 향후 개발 계획

다음 버전에서는 분석 정확도를 향상시키기 위해 다음 기능을 개발할 예정이다.

* 이벤트 점수(Event Score) 기반 추천
* 이동 평균 기반 증가율 계산
* 연속 이벤트 자동 병합
* 최소 채팅량 기준 적용
* 복수 키워드 분석
* 추천 점수 기반 정렬
